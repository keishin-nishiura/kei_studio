# kei_studio

個人開発スタジオ「keistudio」の公式サイト。  
開発の進捗・プロダクト紹介・HP モックアップ一覧を公開しています。

---

## ファイル構成

```
kei_studio/
├── index.html        # トップページ
├── product.html      # プロダクト紹介ページ
├── mocks.html        # HP モック一覧ページ
├── about.html        # About ページ
├── header.js         # 共通ヘッダーコンポーネント
├── main.js           # トップページ用スクリプト
├── projects.js       # プロジェクト一覧スクリプト
├── style.css         # 共通スタイル
├── assets/           # 画像などの静的ファイル
└── mocks/            # HP モックの HTML ファイル置き場
    └── restaurant-corp.html
```

## HP モックの作り方

### 1. `mocks/` フォルダに HTML ファイルを作成する

`mocks/` フォルダの中に新しい HTML ファイルを追加します。

**ファイル名の命名規則（推奨）**

```
クライアント名-種類.html
```

| 例                            | 説明                             |
| ----------------------------- | -------------------------------- |
| `cafe-corp.html`              | カフェのコーポレートサイト       |
| `yoga-lp.html`                | ヨガスタジオのLP                 |
| `photographer-portfolio.html` | フォトグラファーのポートフォリオ |
| `accessory-ec.html`           | アクセサリーのECサイト           |

**カテゴリの種類**

| 値          | 意味               |
| ----------- | ------------------ |
| `corp`      | コーポレートサイト |
| `lp`        | ランディングページ |
| `portfolio` | ポートフォリオ     |
| `ec`        | EC / ショップ      |
| `other`     | その他             |

---

### 2. `mocks.html` の `MOCKS` 配列にデータを追記する

`mocks.html` を開き、`MOCKS` 配列の末尾にオブジェクトを追加します。

```js
{
  id: "mock-007",           // 他と被らない一意のID
  title: "カフェ公式サイト",  // 一覧に表示されるタイトル
  desc: "落ち着いた雰囲気のカフェ向けコーポレートサイト。",  // 説明文
  category: "corp",         // corp / lp / portfolio / ec / other
  tags: ["カフェ", "ライト"], // フリーワードのタグ（配列）
  status: "done",           // new（新着）/ wip（制作中）/ done（完成）
  date: "2026-04-08",       // 作成日（YYYY-MM-DD）
  url: "mocks/cafe-corp.html", // mocks/ からの相対パス（未作成なら null）
},
```

> `url` を `null` にすると「プレビューなし」のプレースホルダーが表示されます。  
> ファイルができたタイミングでパスを設定してください。

---

### 3. 確認する

ブラウザで `mocks.html` を開くと、追加したモックがカード一覧に表示されます。  
カードをクリックするとライトボックスでプレビューが確認できます。

---

## モック HTML の作成ガイドライン

- `mocks/` 内の HTML は **スタンドアロン**（1ファイル完結）で作るのが基本です。
- `header.js` などの共通コンポーネントは**読み込まない**こと（ライトボックス内でヘッダーが二重になるため）。
- 外部フォント（Google Fonts など）や CDN の利用は問題ありません。
- 画像は `../assets/` を参照するか、外部URL（Unsplash など）を使ってください。
- レスポンシブ対応（`@media` クエリ）を入れておくとライトボックスでの見栄えが良くなります。

## ステータスの使い方

| ステータス | バッジ色       | 使うタイミング       |
| ---------- | -------------- | -------------------- |
| `new`      | 青（accent）   | 最近追加したモック   |
| `wip`      | 黄色           | まだ制作途中のモック |
| `done`     | 緑（accent-3） | 完成したモック       |
